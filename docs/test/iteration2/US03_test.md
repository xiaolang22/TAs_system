# US03 测试报告（从简历自动填充档案）

日期：2026-04-12  
测试方式：手工功能测试

## 1. 测试目标

验证 US03「从简历自动填充个人资料」功能是否满足以下验收点：

1. 上传简历后，系统可提取教育背景、技能和经历文本。  
2. 提取结果可预填到个人资料编辑表单。  
3. TA 可在保存前修改预填内容并成功保存。

## 2. 测试环境

- OS：Windows  
- JDK：21.0.2  
- Tomcat：10.1.49  
- Browser：Chrome / Edge  
- 应用地址：`http://localhost:8080/TA_Recruitment/login`  
- 相关页面：`/TA_Recruitment/profile`  

## 3. 前置条件

1. 使用 TA 账号登录系统。  
2. 已存在可编辑的 TA Profile（包含 studentId）。  
3. 准备可上传的简历文件（本次使用 `docx`）。  

## 4. 测试用例与结果

### TC01 - 上传简历后自动提取并预填

**步骤**
1. 进入 Profile 页面。  
2. 在 Upload CV 区域选择简历文件并上传。  
3. 观察页面提示与表单内容。  

**预期结果**
- 页面提示上传成功。  
- 页面提示已完成 US03 预填。  
- `Programme`、`Skills`、`Experience` 字段出现提取后的内容。  

**实际结果**
- 上传成功提示正常显示。  
- 预填提示正常显示。  
- 三个字段均已自动填充对应内容。  

**状态**
Pass

---

### TC02 - 预填内容可编辑并保存

**步骤**
1. 在已预填的 `Skills`、`Experience` 字段中手动修改文本。  
2. 点击 `Save Profile`。  
3. 重新打开当前 TA 的 Profile。  

**预期结果**
- 保存成功提示出现。  
- 修改后的内容被持久化，重新打开后仍为修改值。  

**实际结果**
- 保存成功提示正常。  
- 重新进入页面后显示为已修改内容，数据保存正确。  

**状态**
Pass

---

### TC03 - 与现有 US02 上传能力兼容

**步骤**
1. 在同一 TA 账号下重复上传简历。  
2. 检查当前 CV 文件名显示和后续预填行为。  

**预期结果**
- 文件上传成功，当前 CV 文件名更新。  
- 自动提取与预填流程可重复执行。  

**实际结果**
- 当前 CV 文件名更新正常。  
- 再次上传后仍可成功预填。  

**状态**
Pass

## 5. 测试结论

US03 功能已通过本次手工验证。  
系统可在简历上传后自动提取并预填教育/技能/经历信息，且支持用户在保存前编辑，符合用户故事验收标准。
