# US10 测试报告

## 1. 基本信息

- Story ID：US10
- Story 名称：查看候选人的技能匹配分数和缺失技能
- 测试人：`郭奕轩`
- 测试日期：`2026/04/12`

## 2. 测试目标

验证以下验收标准是否满足：

1. 系统可根据岗位要求和候选人资料/CV数据计算可见的匹配分数。
2. 缺失技能以可读的自然语言列出。
3. 分数和说明显示在候选人评审视图中。

## 3. 测试环境

- OS：`Windows 11`
- JDK：`21.0.2`
- Tomcat：`10.1.49`
- 部署方式：`scripts/deploy.bat`

## 5. 测试范围与方法

- 方法：手工黑盒测试
- 覆盖范围：
  - 正常流程（有匹配）
  - 部分匹配
  - 无匹配
  - 空输入/异常输入

## 6. 测试用例与结果

| 用例ID | 前置条件 | 操作步骤 | 预期结果 | 结论 |
| --- | --- | --- | --- | --- |
| US10-TC01 | 存在至少1个TA档案，skills非空 | MO登录 -> Home -> Go to Candidate Review；输入 `Java, communication, SQL`；点击计算 | 每位候选人显示匹配分数与缺失技能 | Pass |
| US10-TC02 | 候选人只包含部分技能 | 输入 `Java, Python, SQL` | 分数 < 100%，缺失技能列出 `Python` 等 | Pass |
| US10-TC03 | 候选人与要求无交集 | 输入 `Machine Learning, C++` | 分数接近0%，缺失技能完整列出 | Pass |
| US10-TC04 | requiredSkills 为空 | 直接提交空白输入 | 页面提示需输入至少一个技能，不崩溃 | Pass |
| US10-TC05 | 非MO账号（TA）访问 | TA登录后直接访问 `/mo/review` | 返回403或被拒绝访问 | Pass |
