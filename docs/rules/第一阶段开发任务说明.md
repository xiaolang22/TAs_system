# 迭代1、2阶段任务说明

本轮开发聚焦从零实现迭代1与迭代2阶段的开发任务，具体如下：

## 迭代1（Sprint 1）：核心 MVP 基础功能
**时间：2026-03-08 至 2026-03-22**

| Story ID | 名称         | 描述                                                   | 优先级  | 估点   |
| -------- | ---------- | ---------------------------------------------------- | ---- | ---- |
| US01     | 创建申请人档案    | 作为 TA，我希望创建和编辑我的个人档案（包含联系方式、教育背景、技能、空闲时间），以便模块负责人查看。 | Must | 3 SP |
| US02     | 上传简历       | 作为 TA，我希望上传我的简历（PDF/DOC/DOCX），以便在申请工作时展示我的经历。        | Must | 3 SP |
| US04     | 浏览、搜索和筛选工作 | 作为 TA，我希望浏览并筛选可用的 TA 或监考工作，以便快速找到合适的岗位。              | Must | 5 SP |
| US06     | 申请工作       | 作为 TA，我希望针对某个职位提交申请，以便模块负责人审核我。                      | Must | 5 SP |
| US08     | 发布工作       | 作为 MO，我希望创建并发布一个职位，以便合适的 TA 能够申请。                    | Must | 5 SP |

## 迭代2（Sprint 2）：核心流程增强 + AI 基础功能
**时间：2026-03-23 至 2026-04-12**

| Story ID | 名称                | 描述                                                   | 优先级    | 估点   |
| -------- | ----------------- | ---------------------------------------------------- | ------ | ---- |
| US07     | 查看申请状态时间线         | 作为 TA，我希望查看每个申请的状态和时间线（如已提交、审核中、入围、已接受/拒绝），以便了解当前进展。 | Must   | 3 SP |
| US09     | 查看某职位的所有申请人       | 作为 MO，我希望在一个列表中查看某个职位的所有申请人，以便高效比较候选人。               | Must   | 5 SP |
| US11     | 筛选或拒绝申请人          | 作为 MO，我希望更新申请结果（入围、接受、拒绝），以便记录招聘决策并通知 TA。            | Must   | 5 SP |
| US03     | 从简历自动填充档案         | 作为 TA，我希望系统从我的简历中自动提取关键信息（教育、技能、经历），避免手动重复输入。        | Should | 5 SP |
| US10     | 查看候选人的技能匹配分数和缺失技能 | 作为 MO，我希望看到每个申请人的技能匹配分数和缺失技能说明，以便更快筛选。               | Should | 5 SP |
