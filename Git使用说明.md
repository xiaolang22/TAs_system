# Git使用说明

## 首先需要明确Git的三个区：工作区、暂存区、版本库

**工作区** 
- 是你电脑上实际看到的项目目录，即你正在编辑、修改代码的地方。这里的文件可以是未跟踪（新增）、已修改（未暂存）或已暂存（准备提交）状态。
- 通过`git add` 可以将工作区的修改放入暂存区。

**暂存区** 
- 是一个临时存储区域，用于记录下一次要提交的修改。它位于 `.git/index` 文件中，是工作区和版本库之间的缓冲层。你可以有选择地将工作区的部分修改放入暂存区，然后提交。
- 通过`git add` 将工作区变更加入暂存区；`git reset HEAD <file>` 将暂存区内容移除（但保留工作区修改）。

**版本库**
- Git 存储所有提交历史的数据库，每个提交都是一个快照，包含完整的文件内容和元数据。提交后的内容永久保存在版本库中（除非被垃圾回收）。
- `git commit` 将暂存区内容生成一个新提交，存入版本库；`git push` 将本地版本库中的提交同步到远程版本库。

## 开发过程需要用到的Git命令

1. 自己本地没代码的时候先克隆一份远端代码，克隆后在本地工作区会出现main分支对应的代码：
```bash
git clone https://github.com/xiaolang22/TAs_system.git
```
2. 当间隔许久没进行开发的时候，远端main分支可能发生变化，所以需要在每一次开发前在本地拉取一下远端代码，保证当前工作区的代码是最新的：
```bash
git pull
```
3. 每一次编写代码都必须在自己的分支上进行，不允许直接修改main分支并push到远端。所以需要创建/切换到自己的分支：
```bash
git checkout main  # 切换到main分支
git checkout -b <自己的分支名>  # 切换到自己的分支（若分支不存在会自动创建并切换）
```
4. 在自己的分支上开发时，需要用到如下一些命令：
```bash
git status # 查看当前仓库的状态（可以看到有哪些文件没有commit等）
git add <目标文件> # 将修改好的目标文件添加到缓存区
git commit -m "一些提交注释"  # 将暂存区的所有文件提交到版本库（注释必须要写嗷）
git push origin <自己的分支名> # 今天的开发结束了，就可以提交自己本地的代码到远端
```

## 一些约定

- 分支的命名规则：个人英文名/该分支的主要功能，例如`ChenDahong/navigation`
- 自己的分支合并前需要在github上提交Pull Request，并邀请一名成员审查
- 禁止push别人的分支或者直接push main分支（这个后面会加分支保护限制）
